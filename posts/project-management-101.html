Project management 101 for solo builders

Published on 2026-07-01

When you are building projects alone—whether it's an app, a research paper, or a study system—you are both the manager and the executioner. It's incredibly easy to get distracted by shiny new ideas and abandon the old ones.

The core principles

1. Scope small: Define exactly what the "done" state looks like before you start typing code or researching. If it takes longer than 2 weeks, it's too big.

2. Track ruthlessly: Keep a simple checklist. You don't need Jira. A markdown file or a piece of paper is often enough for solo work.

3. Ship it: The project isn't real until someone else can see it. Deploy it, publish it, or print it out.

I'm still learning this, but forcing myself to build small, verifiable projects has been a game changer.
